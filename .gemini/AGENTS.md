# .gemini

## Purpose
Stores managed Gemini CLI configuration generated and maintained by `omg setup`, including sandbox settings, managed guidance notes, and the default subagent catalog.

## Key Files

| File | Description |
|------|-------------|
| `settings.json` | Gemini tool configuration (defaults `tools.sandbox` to `docker`). |
| `GEMINI.md` | Managed setup note block documenting active scope and command guidance. |
| `sandbox.Dockerfile` | Baseline sandbox image used by smoke checks. |

## Subdirectories

| Directory | Purpose |
|-----------|---------|
| `agents/` | Subagent catalog for experimental subagents backend. See `agents/AGENTS.md`. |

## For AI Agents

### Working In This Directory
- Preserve managed marker sections in `GEMINI.md` (`# >>> ... >>>` / `# <<< ... <<<`).
- Prefer running `npm run setup` instead of manual edits when possible.
- Keep sandbox configuration compatible with `scripts/sandbox-smoke.sh` expectations.

### Testing Requirements
- Run `npm run setup -- --dry-run` (or `npm run setup`) after changing setup logic.
- Validate with `bash scripts/sandbox-smoke.sh --dry-run`.

### Common Patterns
- Setup logic performs idempotent writes and avoids destructive overwrites.

## Dependencies

### Internal
- Generated by `src/installer/index.ts` and consumed by `src/team/runtime/subagents-backend.ts`.

### External
- Gemini CLI runtime conventions and Docker sandbox usage.
